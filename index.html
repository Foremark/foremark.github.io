Foremark is currently under development — You might find some features incomplete or broken.

Foremark is a technology for writing a plain-text document which renders nicely on a web browser.

It does not require a plugin, pre-compilation process, or dedicated viewer software.

Why Use Foremark?

  • Text files can be edited by any text editor of your choice. They also work very well with a version control software.

  • Foremark automatically loads assets from CDN, but if needed, it can fully operate without an internet connection. Just use a self-contained bundle (foremark.bundle.js) available here.

  • Created documents can be shared easily as they are self-contained, only need a web browser to display, and adapt to various display devices (like PC, tablet, and paper).

  • The syntax is based on Markdown with some extensions and easy to learn and use.

Get Started

Copy the following code, save it as hello.fm.xhtml, and open it with a web browser. 

<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="en"><body><mf-text><![CDATA[

                            **Title**

Welcome to Foremark. It's a way to write a _stylized_ text document without a fuss.

:::           .----------.
::: Write --> | Document | --> Web Browser
:::           '----------'

]]></mf-text> <!-- Foremark footer -->
<script src="https://unpkg.com/foremark/browser/foremark.js" data-rel="foremark" async="async" />

Credits to Other Projects

  • The formatting of Foremark pulls much inspiration from John Gruber’s Markdown while making many intentional changes.

  • Foremark was heavily inspired by Morgan McGuire’s Markdeep. It also includes a portion of Markdeep.

  • Rendering of an ASCII diagram is performed using a Rust library Svgbob.

  • Displaying LaTeX equations was made possible by a JavaScript library KaTeX.

  • Foremark is built using Webpack. Its code splitting feature is essential for lazy-loading various components.

  • The code highlighting feature is provided by a JavaScript library Highlight.js.
